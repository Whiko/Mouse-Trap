Livrables : rapport (version papier) + code du projet à remettre le 7 mai

Indications concernant le rapport :

En plus de ce qui est spécifié dans le document décrivant les deux sujets, 
le rapport de projet peut être organisé de la façon suivante :

1. Rappel du sujet : expliquez ce que vous en avez retenu, vos adaptations,
vos hypothèses, dégagez les fonctionnalités principales

2. Analyse : 

2.1 Donnez l'architecture générale de votre application : organisation des
threads, des communications, faites des schémas

2.2 Pour chaque fonctionnalité, expliquez comment vous la réalisez, précisez les 
algorithmes que vous allez utiliser, mettez en evidence comment vous résolvez les 
problèmes de concurrence, de communication (protocole), les autres concepts 
des systèmes d'exploitation que vous utilisez.

3. Implémentation :
Décrivez les packages, les classes et leur orgnisation
Ne pas mettre tout le code mais des extraits commentés.

4. Tests et utilisation
Expliquez commeent fonctionne votre programme au moyen d'exemples,
de copies d'écran etc.

5. Conclusion et évolutions

6. Bibliographie (livres ou sites web que vous avez consultés)

7. Annexes éventuelles
